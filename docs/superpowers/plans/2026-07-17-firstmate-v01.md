# PLAINOPS v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A local-first AI DevOps agent ("PLAINOPS") a founder runs on their own laptop: connect a repo + AWS account + Anthropic API key, chat in plain English, approve a cost-estimated plan, and get a live containerized web app deployed into *their own* AWS account ‚Äî credentials never sent to the model.

**Architecture:** A single Node/TypeScript process serves a localhost dashboard (vanilla JS + SSE) and runs an agent loop (direct Anthropic SDK tool-loop ‚Äî we own the harness). The agent's tools do the real work locally: analyze repo ‚Üí render an OpenTofu blueprint (ECS Fargate + ALB + optional RDS) ‚Üí estimate cost from a pricing table ‚Üí on human approval, `tofu apply`, push source zip to S3, build the image with AWS CodeBuild (no local Docker), roll the ECS service, health-check the URL. Secrets flow founder ‚Üí local encrypted vault ‚Üí AWS Secrets Manager via SDK; the model only ever sees `{{secret:NAME}}` placeholders.

**Tech Stack:** Node ‚â•20, TypeScript (ESM, run via tsx ‚Äî no build step), vitest, express (localhost UI + SSE), `@anthropic-ai/sdk` (BYO key, default model `claude-sonnet-5`), `@aws-sdk/client-{sts,s3,ecs,codebuild,cloudwatch-logs,cost-explorer,secrets-manager}`, `archiver` (zips), `adm-zip` (unzip tofu download), OpenTofu binary (system PATH or auto-download, `PLAINOPS_TOFU_PATH` override).

## Global Constraints

- Windows 10 is the primary dev/test machine: spawn processes with `execFile`/`spawn` + arg arrays (never shell strings); all paths via `path.join`; no native Node modules (no keytar).
- No local Docker dependency anywhere ‚Äî images build in the founder's account via CodeBuild.
- Secret values must NEVER appear in: model requests, tofu variables/plan/state, logs, audit trail, or SSE payloads. Only `{{secret:NAME}}` placeholders. Values reach AWS only via `PutSecretValue` over the AWS SDK.
- Mutating actions (`provision_infra`, `deploy_app`, `set_app_secret`, `destroy_infra`) are gated by a code-enforced PendingAction approval (dashboard click). The model cannot bypass; timeout 15 min ‚Üí auto-reject.
- Every AWS resource tagged `managed-by=plainops`, `plainops-project=<name>`.
- One region per project (default `us-east-1`, configurable at project create) used consistently for S3/ECR/CodeBuild/ECS/CE.
- All chat/tool text destined for the model or logs passes through the scrubber first.
- App data lives in `~/.plainops/` (config.json, vault.enc, vault.key, state.json, audit.log, projects/<name>/tf/).
- Estimates labeled "estimate ¬±15%"; blueprint avoids NAT gateways (public subnets + strict SGs) to keep base cost ‚âà $30‚Äì45/mo.

## Failure-Mode Analysis ‚Üí Design Mitigations (veteran pass)

| # | Failure mode | Mitigation baked into design |
|---|---|---|
| 1 | Founder laptop has no Docker (or WSL-only Docker) ‚Üí local image build fails | Images built by **CodeBuild** in their account from an S3 source zip; laptop never builds |
| 2 | NAT gateways silently add ~$65/mo | Blueprint uses public subnets + SG lockdown (svc only reachable from ALB SG); no NAT anywhere |
| 3 | Secrets leak into tfstate/plan | Tofu creates empty Secrets Manager shells only; values set via SDK `PutSecretValue`; task def references ARNs |
| 4 | Secrets leak into model context via command output (e.g., env echo) | Central scrubber replaces every vault value (len‚â•6) + AKIA-style patterns in ALL tool results before they reach the model, SSE, or logs |
| 5 | Model "approves itself" / prompt injection from repo files triggers mutation | Approval is a code path (HTTP POST from dashboard) resolving a PendingAction; tool handler blocks until human click; repo content is treated as data |
| 6 | Local tfstate lost ‚Üí orphaned infra | After every apply/destroy, timestamped state backup uploaded to the bootstrap S3 bucket; `destroy` also available via dashboard any time |
| 7 | Surprise AWS bill | Cost estimate before approval; AWS Budget with email alert (optional param); everything tagged; one-click destroy; smallest instance defaults |
| 8 | OpenTofu download blocked/version 404 | Prefer `tofu`/`terraform` already on PATH; else try pinned version list; else clear error + `PLAINOPS_TOFU_PATH` override instructions |
| 9 | Windows quoting/path bugs | `execFile` with arg arrays everywhere; tests run on Windows first |
| 10 | Cost Explorer has ~24h data delay / not enabled | Costs panel degrades to "No billing data yet (AWS delays up to 24h)" instead of erroring |
| 11 | Deploy hangs (build or ECS never stabilizes) | CodeBuild poll timeout 15 min; ECS wait timeout 10 min with service events surfaced to chat; failure returns actionable logs |
| 12 | Founder has no Anthropic key / AWS creds | Preflight endpoint checks node/git/creds/key/tofu and dashboard onboarding walks through fixes before chat unlocks |
| 13 | Giant zip uploads (node_modules) | Zip excludes `.git`, `node_modules`, `dist`, `.next`, `__pycache__`, `.venv`; warn >50 MB, hard stop >200 MB |
| 14 | `:live` tag race / stale image | CodeBuild pushes `:live` + `:<timestamp>`; ECS forceNewDeployment after successful build only; timestamps enable manual rollback |
| 15 | Concurrent mutations corrupt state | Single global action lock: one provision/deploy/destroy at a time; UI disables buttons while locked |
| 16 | Dashboard unverifiable without AWS during development | `PLAINOPS_DEMO=1` boots the UI with faked state/events for smoke-testing UI end-to-end |
